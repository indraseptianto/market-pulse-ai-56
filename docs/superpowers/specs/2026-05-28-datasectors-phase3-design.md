## 1. Portfolio Tracker

### 1A. Data Input
- **Manual Entry** — Input transaksi: BUY/SELL, symbol, quantity (lots), price, date
- **CSV Import** — Upload broker CSV (Sinarmas, Stockbit, Bareksa). Auto-detect column mapping.
- **Storage** — `localStorage` key `stratum_portfolio`. Schema:
```typescript
interface Position {
  symbol: string;
  avgBuyPrice: number;
  totalLots: number;
  realizedPnL: number;
  dividendsReceived: number;
  transactions: Transaction[];
}
interface Transaction {
  type: "BUY" | "SELL" | "DIV";
  date: string;
  lots: number;
  price: number;
  total: number;
}
```

### 1B. Metrics (Essential Only)
- Total portfolio value = Σ(position.lots × current_price × 100)
- Cost basis = Σ(position.lots × avgBuyPrice × 100)
- Unrealized P&L = Total value - Cost basis (IDR + %)
- Allocation pie chart (Recharts PieChart) per symbol
- Per-position: lots, avg price, current price, P&L, P&L%

### 1C. Pages & Components
- `/portfolio` — Main dashboard: KPI row (total value, P&L, positions count), allocation pie, position table
- `src/components/portfolio/PositionTable.tsx` — Table with P&L color coding
- `src/components/portfolio/AllocationChart.tsx` — Pie chart with legend
- `src/components/portfolio/TransactionModal.tsx` — Buy/Sell/Dividend entry form
- `src/components/portfolio/CsvImport.tsx` — File upload + column mapper

## 2. Sector Analysis

### 2A. Sector Heatmap
- Treemap using Recharts `Treemap` component
- Data: sector name, total market cap, avg change_pct
- Color: green scale for positive, red scale for negative (d3-like gradient)
- Size: proportional to total market cap
- Click sector → navigate to sector detail (filter equities by sector)

### 2B. Sector Rotation Tracker
- Multi-timeframe performance table: 1D, 1W, 1M, 3M, 6M, 1Y
- Sort by selected timeframe
- Momentum score = weighted composite (recent weight > older)
- Top/Bottom 3 sectors highlighted

### 2C. Radar Chart Comparison
- Compare 2-4 sectors side-by-side
- Axes: Avg P/E, Avg Dividend Yield, Avg ROE, Market Cap Weight, Momentum Score
- Recharts `RadarChart` with `PolarGrid`, `PolarAngleAxis`, `Radar`

### 2D. Pages
- `/sectors` — Sector heatmap + rotation tracker + top/bottom sectors
- `/sectors/$sectorName` — Equities filtered by that sector

## 3. Stock Screener v2

### 3A. Multi-Factor Filter
Filters:
- RSI: range slider (0-100), preset: <30 oversold, >70 overbought
- Dividend Yield: min % input
- Market Cap: dropdown (Large >50T, Mid 10-50T, Small <10T)
- Sector: multi-select dropdown
- P/E Ratio: max input
- P/BV Ratio: max input
- Price Change: >X% movers

### 3B. Composite Score
- Score per stock = weighted sum:
  - RSI_score (0-100, lower is more oversold = higher score)
  - DivYield_score (normalized 0-100)
  - Momentum_score (normalized by change_pct)
- Sort by composite score descending
- Display score badge per result

### 3C. Results
- Table: Symbol, Name, Price, Chg%, RSI, DivYield%, P/E, P/BV, Score
- Click row → navigate to stock detail
- Pagination (20 per page)

## 4. Automated Daily Report

### 4A. Telegram Text Report (jam 7 pagi)
- Cron job every day at 07:00
- Format: emoji + markdown
- Content:
  - Market Overview: top 5 gainers, top 5 losers
  - Sector Snapshot: sector with best/worst performance today
  - Upcoming Dividends: stocks going ex-date this week
  - Watchlist Summary: current P&L of watchlist stocks
  - AI Market Brief: 2-3 sentence market commentary via AI

### 4B. HTML Report (stored locally)
- Stored at `/opt/daily-report/index.html`
- Served via port 8788
- Content:
  - Portfolio snapshot (mini equity curve, top positions)
  - Sector heatmap (static image)
  - Top movers (gainers + losers, top 20)
  - Dividend calendar this week
  - News highlights
- Generated by cron script at 07:00
- Accessible at `http://43.133.150.19:8788/`

## 6. Technical Notes

- Portfolio data in `localStorage` (no backend needed)
- Sector data from DS API: `getSectorPerformance`, `getSectorDetail`
- Screener data from DS API: `getStockEquitiesV2` (bulk, client-side filter)
- HTML report uses Jinja2-style template written as Python string
- Telegram report uses Hermes `send_message` via API or direct curl
- Both report scripts read from DS API for latest market data
